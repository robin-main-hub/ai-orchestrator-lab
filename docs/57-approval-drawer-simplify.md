# 57 — 승인 드로어 단순화 (UX 개선 #2)

매일 수십 번 누르는 Control Queue 드로어인데, 카드마다 버튼 6개(승인/질문/수정/
위임/차단/보관)가 3열 그리드로 빽빽했다. 대부분은 "승인" 하나만 누르는데 매번 6개
중 골라야 하는 액션 피로.

## 1버튼 기본 + 더보기 (회귀 0)

- **접힌 카드(기본)**: 큰 "승인" 버튼 하나 + "⋯ 더보기" 토글. 더보기를 누르면 나머지
  5개(질문·수정·위임·차단·보관)가 기존 3열 그리드로 펼쳐지고 "접기"로 되돌린다.
  상단 lane 필터로 특정 동작을 고른 경우는 그 동작만 단독 표시(기존 동작 유지).
- **키보드 단축키 시각화**: `⌘⏎`(첫 항목 승인)은 이미 있었지만 드로어 안에서 안
  보였다. 첫 항목 승인 버튼에 `⌘⏎` kbd를 붙이고, 푸터에 `⌘⏎ 첫 항목 승인 · ⌘⌫
  거절 · esc 닫기`를 노출. `⌘⌫`(첫 항목 거절)는 ⌘⏎의 대칭으로 새로 추가
  (useGlobalShortcuts onReject → handleResolveNextPermission("rejected"); 입력창
  안에서는 Backspace 삭제와 충돌하지 않게 제외).
- **위험 명령 강조 + 확인 스텝**: `isRiskyApprovalItem`(순수: sourceTrust ===
  "untrusted")인 항목은 빨간 테두리 + "위험" 배지 + 승인이 2스텝("승인" →
  "정말 실행?")이 된다. 일반 명령과 같은 한 번 클릭으로 통과되지 않는다. 실제
  sourceTrust에서만 도출 — 가짜 위험 표시 없음.

## 원칙

- **빈도순 단순화**: 99%인 "승인"을 1클릭, 나머지는 한 단계 뒤로.
- **숨은 단축키를 보이게**: 이미 있던 키보드 경로를 UI에 드러내고 거절 대칭 추가.
- **위험은 다르게**: untrusted 실행만 확인 스텝 — 거짓말하지 않는 위험 신호.
- **회귀 0**: 6개 동작·핸들러·lane 필터 모두 유지(더보기로 접었을 뿐). 위험 판정은
  순수 함수로 분리 검증. typecheck·프로덕션 빌드 통과, desktop 1116 그린(+4).
