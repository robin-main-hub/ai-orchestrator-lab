# 65 — 대시보드 오늘의 파티 동적화 + 클릭 (기능 개편 F#4)

대시보드 "소환진 — 오늘의 파티"가 App.tsx에 쿠루미/유노 2명으로 하드코딩돼 날짜가
바뀌어도 그대로였고, 카드를 클릭해도 아무 반응이 없었다(바로 아래 도감 카드는 클릭
가능했는데도).

## 한 일 (회귀 0)

- **날짜 로테이션 + 실신호 선발**: `selectDailyParty`(순수·테스트)가 오늘 날짜 시드로
  PERSONA_CODEX를 회전시키되, **지금 활성**(Hermes 슬롯 바인딩)·**최근 작전**(run
  history) 페르소나를 앞세운다. App이 `eventLog`(최근)와 `loadHermesPool().slots`
  (활성), 오늘 날짜로 호출해 3명을 뽑는다. 날이 바뀌면 추천이 돌고, 활동이 있으면
  반영된다.
- **왜 오늘인가(reason)**: 각 멤버에 "오늘 활성 / 최근 작전 / 오늘의 추천" 배지를
  카드 좌상단에 — 정적 쇼케이스가 아니라 "왜 이 캐릭터가 오늘 파티에 있는지"를 보여준다.
- **클릭 상호작용**: 파티 `figure`를 도감 섹션과 같은 `<button onClick>` 패턴으로
  감싸 클릭 시 `buildCodexDetail` → `PersonaCodexModal`(소환·대화 동선 포함)을 연다.
  기존 도감 모달을 그대로 재사용.

## 원칙

- **거짓말하지 않는다**: 로테이션은 날짜 시드로 결정적이되, "활성/최근"은 실제 Hermes
  슬롯·run history에서만. reason이 진짜 신호를 가리킨다.
- **기존 동선 재사용**: 클릭→상세 모달은 도감과 동일(buildCodexDetail/PersonaCodexModal).
- **회귀 0**: DashboardPersona에 `reason?` 옵셔널 추가, 선발은 순수 함수(+5 테스트).
  desktop 1132 그린(+6), 빌드 통과.
