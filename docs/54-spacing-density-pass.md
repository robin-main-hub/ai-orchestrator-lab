# 54 — 여백·밀도 패스 (디자인 정리 4탄)

디자인 리뷰 지적: "밀도 vs 여백 — 빡빡하고, 폰트가 다 비슷한 크기라 위계가 없다."

## 근본 원인: spacing 스케일이 없었다

정찰 결과 `tokens.css`는 색·폰트·radius만 정의하고 **간격 토큰이 0개**였다.
`styles.css`(약 9,900줄)에 `padding`/`gap`이 바레 px로 ~670곳 하드코딩돼, 패널마다
10px/8px/12px이 제각각 — 그래서 어떤 카드는 빡빡하고 어떤 건 헐겁게 드리프트했다.
폰트도 본문·메타·제목이 대부분 11~13px 좁은 대역에 몰려 위계 신호가 약했다.

## 4px 리듬 토큰 + 위계 3단 (tokens.css 신설)

```css
--space-1..6: 4 8 12 16 20 24px   /* 공유 간격 스케일 */
--fs-meta/body/label: 12 / 13 / 14px  /* 제목·본문·메타 분리 */
```

추가만으로는 효과 0(미사용 토큰). 가장 빡빡하고 자주 보이는 레거시 BEM 패널부터
점진 적용 — `tokens.css`가 본래 표방하는 단계적 마이그레이션 방식 그대로.

## 이번에 적용한 곳 (densest first)

- 좌/우 레일·미니패널·rail-panel·center-board: `padding 10px → --space-3(12)`,
  헤더 `gap 8 → --space-2`. 패널 내부에 숨 쉴 틈.
- 페르소나 카드: `padding 10/12 → --space-3`, 카드 간 `margin 8 → --space-3`,
  스탯 `margin-top 8 → --space-3`.
- 미션보드: 카드 간 `gap 10 → --space-3`, 카드 `padding 10/12 → --space-3/--space-4`,
  내부 `gap 6 → --space-2`. 제목 `13 → --fs-label(14)/600`, 목표 `12 → --fs-body(13)`
  — 제목이 메타에서 분리돼 카드가 읽힌다.

콕핏 카드는 별도 시스템(Tailwind gap-3/4)이라 이번 패스 대상이 아님 — 콕핏은 이미
3단계 드릴다운(docs/51)으로 정리됨.

## 원칙

- **한 리듬**: 흩어진 px 대신 4px 스케일 토큰으로 패널 밀도를 고르게.
- **위계 회복**: 제목/본문/메타를 다른 폰트 rung으로 분리.
- **점진·회귀 0**: CSS만(로직 0), 가장 빡빡한 패널부터. 토큰은 이후 사이트들이
  점진 채택. 프로덕션 빌드 통과, desktop 1094 그린.
