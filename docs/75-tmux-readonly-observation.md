# 75 — Tmux 스웜 보드를 읽기 전용 관측 보드로

스웜 작업대는 "개별 에이전트에게 명령하는 곳"이 아니다 — 지시·수정은 대화 탭에서
상대를 바꿔가며 한다. 여기서는 **에이전트들이 지금 뭘 하고 무슨 결론을 냈는지 한눈에**
보기만 하면 충분하다. (UX#5에서 넣었던 per-pane 명령 입력을 의도에 맞게 거둬들인다.)

## 한 일

- **카드 = 읽기 전용 관측 카드**: per-pane 명령 입력(input·전송)을 제거하고, 각 카드에
  **최근 결론/답변 한 줄**(`latestConclusionLine` — 마지막 비어있지 않은 출력 줄)을
  추가. 아바타 + 상태 칩(실시간 활동) + 결론을 한눈에. 클릭하면 아래 풀와이드에서 전체
  출력/타임라인.
- **상세 = 읽기 전용**: 풀와이드 상세의 명령 composer를 "읽기 새로고침"(capture)만 남긴
  관측 바로 교체. 출력·타임라인·승인 게이트는 그대로(읽기/승인은 관측·결정이지 명령 아님).
- **명확한 안내**: 상단에 "관측 전용 — 지시·수정은 대화 탭에서 상대를 바꿔가며 하세요".

## 대화창 쪽 스웜 관측 (이미 있음)

다수 에이전트가 움직일 때 대화창에서 무슨 일을 하는지 보는 건 이미 일부 동작한다:
대화 워크벤치 좌측 **에이전트 레일(docs/56, UX#1)**이 활동 상태 도트(작업 중/승인 대기/
막힘)를 보여주고, 우측 사이드 패널 "에이전트" 모드(AgentsSidebar)가 로스터·활동을 보여준다.
TMUX 보드는 그보다 **더 많은 정보(결론·출력)를 한 화면에** 모으는 역할로 분리됐다.

## 원칙

- **역할 분리**: 명령은 대화, 관측은 TMUX. 같은 일을 두 곳에서 하지 않는다.
- **회귀 0**: pane 단위 상태·핸들러는 그대로(detail이 재사용), 카드만 읽기 전용으로.
  typecheck·빌드·desktop suite(1141) 그대로.
