# 66 — 작전극장(Summon Theater) 기능 개편 (기능 개편 F#5)

"작전극장은 뭐하는 곳이냐, 에이전트 클릭해도 반응 없고 이름만으론 알 수가 없다"는
지적. git 이력을 보니 원래 의도가 분명했다 — 다만 그 의도가 화면에 드러나지 않았다.

## git 이력으로 본 원래 의도

2026-06-11 두 커밋: `#481`(work theater phase 1 — character + 6-stage pipeline,
data-wired)이 `lib/workTheater.ts`(deriveTheaterRows 등)와 "누가 어느 단계서 무슨
일을 하는지 1화면, 클릭 가능"한 임베드 패널을 만들고, `#482`(Summon Theater full
screen, v0 Cyber-Neon, REAL data)가 풀스크린 `SummonTheater`로 이식했다. docs/46
원칙: "UI의 멋있는 theater 상태와 실제 관측 상태를 분리한다"(TruthStatus).

문제는 풀스크린 이식 때 **(a) 카드가 어느 단계서 뭘 하는지를 안 보여주고, (b)
클릭 핸들러가 빠졌다**는 것. 화면은 실데이터(WorkItem 단계)에 배선돼 있었지만, 그
데이터가 카드에 안 떴고 상호작용이 없었다.

## 한 일 (의도 복원, 회귀 0)

- **카드에 "어느 단계 · 무슨 일" 표시**: 각 SummonEntry에 `stageKo`(분류→판단→실행→
  대기→승인→완료) + `stageState`(진행/승인 대기/막힘/완료/대기) + `task`(위임 카드
  제목)를 실어, 카드마다 "지금 이 에이전트가 어느 단계에서 무슨 일을 하는지"를 칩+
  한 줄로. `TheaterRow`(실 WorkItem 파생)의 `stageIndex`/`title`을 그대로 사용 —
  이미 있던 실데이터를 비로소 보여준다.
- **클릭 → 그 에이전트와 대화**: 배정된 카드와 중앙 주인공 초상을 클릭/Enter하면
  `onOpenAgent(agentId)` → `handleOpenDelegatedAgentConversation`로 그 에이전트
  대화를 연다. 데모 파티(실 에이전트 없음)는 비클릭(가짜 동선 없음).
- **페이지가 뭔지 설명**: 헤더에 "지금 누가 어느 단계에서 무슨 일을 하는지 한
  화면으로. 카드를 누르면 그 에이전트와 바로 대화" 부제 — 이름만으론 모르던 용도를
  명시.

## TruthStatus 분리 유지 (정직하게)

docs/46 원칙대로 연출과 실관측을 섞지 않는다. 회전 마법진·숨쉬기·타자기 커맨드라인은
연출로 남기고, 카드 단계/작업/클릭 동선은 실 WorkItem·실 에이전트에서만 온다. 데모
파티는 단계 "대기"로 표시하고 클릭 불가 — 가짜 진행/가짜 동선 없음. 타임라인
스크러버의 양방향 되감기(스크럽→화면 시점 변경)는 시점별 상태 재구성이 필요해 이번엔
보류(연출용 타임라인으로 유지) — 별도 후속.

## 원칙

- **의도를 화면에**: 이미 배선된 실데이터(단계·작업)를 카드에 드러내고 클릭을 연결.
- **거짓말하지 않는다**: 실 에이전트만 클릭 가능, 데모는 대기 표시.
- **회귀 0**: lib/workTheater.ts 코어 그대로, SummonTheater에 표시/상호작용만 추가
  (+3 테스트). desktop 1135 그린(+3), 빌드 통과.
