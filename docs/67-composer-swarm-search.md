# 67 — 대화 컴포저 "+" 스웜 서치 탭 (기능 개편 F#2)

다른 AI앱이 MCP를 고르듯, 대화 입력창 "파일 첨부" 옆에 "+" 도구 탭을 만들고
"스웜 서치"를 고르면 대화 내용에 따라 4~16명의 요원이 자동으로 병렬 조사하게.

## 정찰이 드러낸 빈틈

리서치 스웜은 이미 있었지만 **대화창과 완전히 분리된 별도 nav 뷰**였고, 인원은 6명
고정(수동 ±)이며 "비우면 자동 분담"은 빈 임무를 제네릭 한 줄로 채우는 폴백뿐 —
**"대화 내용 → 4~16명 동적 인원 + 역할별 임무"** 로직은 없었다.

## 한 일 (회귀 0)

- **자동 편성기 `planConversationSwarm`**(순수·테스트, 신규): 주제의 무게(단어 수 +
  "비교/설계/보안/트레이드오프…" 신호)로 인원을 4~16명 동적 산정(`recommendSwarmSize`),
  각 요원에게 역할별 조사 측면(광역 탐색/도메인 심층/교차 검증/적대적 반박/리스크
  정량화/트레이드오프 비교/최신성 감시/최종 합본 …)을 자동 분담. 실 페르소나(도감)에
  역할을 매핑. **실제 조사는 기존 LLM 러너가 수행하고, 여기서는 "몇 명을·누구에게·어떤
  측면을"의 결정적 편성만** 한다(서버 비의존, 테스트 가능).
- **컴포저 "+" 도구 메뉴**(MCP 스타일 Popover): 첨부 버튼 옆 "+" → "스웜 서치" 항목.
  선택 시 `onStartSwarmSearch(현재 입력)` 호출. 도구 메뉴라 향후 다른 도구 확장 가능.
- **대화 → 리서치 자동 편성 배선**: App `handleStartSwarmSearch`가 입력(또는 직전 user
  메시지)을 주제로 잡아 `planConversationSwarm`으로 4~16명을 편성하고, 리서치 뷰에
  `seed`로 프리필(주제 + 요원 명단) 후 그쪽으로 이동. 리서치 컨테이너는 새 `seed.id`마다
  주제·요원을 1회 적용하고 "N명 자동 편성됨 — 스웜 배치로 시작" 안내.

## 설계 선택 (정직하게)

메뉴 클릭만으로 4~16개 LLM 호출을 자동 발사하지 않는다 — 주제·요원을 **편성해 두고**
사용자가 "스웜 배치"로 시작한다(비용/의외성 방지). "대화 내용에 따라 자동으로 병렬
일하게 설정을 바꾼다"는 요구를 안전하게: 셋업을 자동으로 채우고, 발사는 명시적으로.

## 원칙

- **대화에서 바로**: 별도 뷰로 가서 6명 고정 수동 설정하던 것을, 대화창 "+"에서
  대화 주제로 4~16명 자동 편성.
- **거짓말하지 않는다**: 편성은 결정적 휴리스틱(테스트), 실제 조사는 LLM 러너. 자동
  발사 없음.
- **회귀 0**: 리서치 컨테이너는 seed prop만 추가(기존 6명 시드 유지), 컴포저는
  onStartSwarmSearch 있을 때만 "+" 노출. desktop 1141 그린(+6), 빌드 통과.
