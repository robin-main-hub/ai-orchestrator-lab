# 64 — 토론 창 스크롤 + 입장 변화 관찰성 (기능 개편 F#3)

토론(Stage3DebateTable) 창을 키우지 않으면 발언 목록(아래쪽)이 잘려 안 보였고,
"누가 어떤 발언으로 설득됐는지"를 입장 궤적 점만 보고는 알 수 없었다.

## 1. 스크롤 수정 (flexbox min-height 버그)

발언 목록 영역은 이미 `flex-1 overflow-y-auto`였지만, 세로 flex 체인에 `min-h-0`가
없었다. flexbox에서 `flex:1` 자식의 기본 `min-height:auto`는 콘텐츠보다 작게 줄지
못해, 헤더(결정레일+의장결정+입장궤적+라운드탭)가 쌓이면 본문이 콘텐츠 높이만큼
늘어나 `overflow-y:auto`가 발동하지 못했다 → 하단 발언이 뷰포트 밖으로 밀려 스크롤도
안 생김.

수정: 외곽 `section`과 스크롤 `div`에 `min-h-0` 두 곳만 추가. 휠 세로 스크롤이 살아난다.

## 2. 입장 점 → 발언 점프 + 전향 강조

"일관된 지지 / N회 입장 변화 / 최종 반대"의 점(●)이 정적이라 관찰이 불가능했다.

- `AgentStancePoint`에 `utteranceId`와 `changed`(직전 결정 입장에서 극성이 뒤집힌
  지점) 추가 — `deriveStanceTrajectories`(순수). 어떤 발언이 입장을 바꿨는지를
  데이터로 잇는다.
- 각 점을 **클릭 가능한 버튼**으로: 클릭하면 그 발언으로 스크롤 + 1.6초 노란 플래시로
  강조(`debate-utterance--flash`, reduced-motion 가드). 발언 `<li>`에 앵커 id 부여.
- **전향(설득)된 발언**은 노란 링(`ring-amber-300`)으로 표시 — "여기서 입장이 바뀌었다"가
  한눈에. 범례: "◎ 노란 링은 입장이 뒤집힌(설득된) 발언".

이제 점을 따라가며 "반대→반대→(여기서)지지"를 클릭해 그 발언의 논리를 직접 읽고,
공감하거나 반박할 수 있다.

## 원칙

- **관찰 가능하게**: 결과(입장 변화 횟수)만이 아니라 그 변화를 일으킨 발언으로 잇는다.
- **거짓말하지 않는다**: changed는 실제 극성 뒤집힘에서만, utteranceId는 실제 발언 id.
- **회귀 0**: 스크롤은 className 2곳, stance 필드는 순수 함수 확장(+테스트). 상위
  컨테이너 CSS는 손대지 않음. desktop 1126 + agents 5 그린, 빌드 통과.
