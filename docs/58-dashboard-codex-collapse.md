# 58 — 대시보드 도감 접기 + 작전 타일 상단 (UX 개선 #3)

대시보드 첫인상이 "소환진 → 도감 18장 → 작전 타일 → 최근 기록"으로 길게 늘어나,
도감 18장이 화면 대부분을 차지하고 실제 행동(작전 타일)이 스크롤 아래로 밀렸다.

## 행동을 위로, 쇼케이스는 접어서 (회귀 0)

- **작전 타일을 첫 시야로**: 섹션 순서를 재배치 — `작전 개시`(타일)를 소환진·도감
  위로 올려 "지금 뭘 할 수 있는가"가 먼저 보인다.
- **도감을 접힌 캐러셀로**: `.dashboard__codex`를 기본 가로 스크롤 캐러셀
  (`is-carousel`: flex + overflow-x + scroll-snap, 카드 고정폭 188px)로. 18장이 한
  줄로 접혀 화면을 잡아먹지 않고, "전체 보기" 토글로 기존 그리드(`is-expanded`)로
  펼친다. 18장은 항상 DOM에 있어(스크롤만) 도감 카피/이름 검사 테스트 유지.
- **"다음 할 일" + 히어로를 한 덩어리로**: 둘을 `.dashboard__top`(작은 gap)으로 묶어
  같은 글래스 비주얼이 하나의 상단 블록으로 읽힌다.

## 원칙

- **행동 우선**: 정보 쇼케이스보다 "할 수 있는 일"을 첫 화면에.
- **접어서 숨 쉴 공간**: 18장 그리드 → 1줄 캐러셀(+펼치기). 데이터/마크업은 그대로,
  컨테이너 CSS만 교체.
- **회귀 0**: PersonaCard/PersonaCodexModal/buildCodexDetail 등 데이터 파이프라인
  무변경. dashboard__next 클래스·도감 타이틀 카피 보존. typecheck·빌드 통과,
  desktop 1117 그린(+1).
