# 제품 브리프

## 한 줄 정의

AI Orchestrator Lab은 맥북 데스크톱 앱에서 여러 AI 모델, 여러 CLI 에이전트, 여러 가상 에이전트를 동시에 지휘하고, 토론 결과를 실제 코딩 실행으로 연결하는 개인용 AI 작업 지휘실이다.

## 왜 필요한가

일반 채팅 앱은 한 모델과 대화하는 데 강하지만, 실제 작업에서는 여러 관점이 필요하다. 설계자는 큰 구조를 보고, 구현자는 코드를 쓰고, 리뷰어는 결함을 찾고, 검증자는 강한 모델이나 로컬 모델로 최종 판단을 한다. 이 앱은 그 흐름을 하나의 작업판에 묶는다.

## 제품 원칙

- 데스크톱 오케스트레이터가 항상 앞에 있다.
- 토론은 독립 기능이 아니라 의사결정과 코딩을 위한 중간 단계다.
- 모든 실행은 기록, 재현, 비교, 전달이 가능해야 한다.
- 원격 서버가 강할수록 좋지만, 서버가 끊겨도 로컬 기능은 살아 있어야 한다.
- API 키와 모델 선택은 고정 설정이 아니라 작업마다 바꿔 끼울 수 있는 프로파일이어야 한다.
- 하나의 모델만 있어도 여러 에이전트 페르소나를 만들어 실용적인 합의 과정을 만들 수 있어야 한다.

## 주요 사용자 흐름

1. 사용자가 작업 목표를 적는다.
2. 오케스트레이터가 작업 유형을 고른다: 토론, 코드 작성, 리뷰, 리서치, 계획, 실행.
3. 필요한 프로바이더 프로파일과 모델을 선택한다.
4. 에이전트들이 역할별로 의견을 낸다.
5. 오케스트레이터가 합의안, 반대 의견, 리스크, 실행 패킷을 만든다.
6. 사용자가 결과를 코딩 에이전트나 터미널 세션에 전달한다.
7. 실행 로그, 결정 이유, 파일 변경, 검증 결과가 하나의 세션으로 저장된다.

## 성공 기준

- 모델을 바꿔가며 토론/검증/코딩을 실행할 수 있다.
- DGX 서버가 연결되면 강한 원격 실행을 사용하고, 끊기면 로컬 모드로 자동 축소된다.
- 사용자는 토론 결과를 복붙하지 않고 코딩 작업으로 바로 넘길 수 있다.
- 여러 API 키와 여러 리셀러 형식의 환경변수 입력을 프로파일로 저장할 수 있다.
- 모든 세션은 나중에 다시 열어 맥락, 결정, 메모리 호출 내역을 확인할 수 있다.
