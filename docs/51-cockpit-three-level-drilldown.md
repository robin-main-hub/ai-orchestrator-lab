# 51 — 관제판 3단계 드릴다운 (디자인 정리 1탄)

디자인 리뷰에서 가장 먼저 지적된 곳: 관제판(콕핏)의 정보 과부하. 화면을 켜면
헤더 배지 5개 + 타일 4개 + 커맨드덱 + 액션 스트립 + 카드 2개가 한꺼번에 쏟아져,
운영자가 "지금 뭘 해야 하지?"를 3초 안에 못 찾는다.

## 3단계로 재구성 (기능 회귀 0)

기존 본문·데이터·액션은 한 줄도 버리지 않고 "위에 얹고 접는" 방식:

```text
L1 (첫 눈, 기본) — 건강 히어로
  전체 신호 한 줄(red/yellow/green) + "지금 처리할 일 하나"의 CTA + 전체 현황 펼치기
L2 (펼침)        — 전체 현황: 글랜스 타일 / 커맨드덱 / 워커·승인 카드 (기존 본문)
L3 (또 펼침)     — 작전 세부: 성숙도 / 영수증 / 핸드오프 / 디스패치 (기존 그대로)
```

- `lib/cockpitHealthRollup.ts`(순수): red/yellow/green + 가장 긴급한 액션(high→
  warning→normal) + 신호 요약. red = 차단 워커·high 액션·고위험 승인·DGX 미러
  오프라인. yellow = 대기 승인·폴백 활성. green = 처리할 것 없음.
- `CockpitHealthHero`(L1): 상태 점 + headline + top action CTA + 펼치기 토글.
- OperatorCockpit: `expanded` 상태로 기존 본문을 L2로 접고 히어로를 항상 표시.
  팔레트 deep-link(initialFocus)나 defaultDetailsOpen이면 자동 펼침 — 기존
  진입 경로 보존.

## 원칙

- **정보 나열 → 다음 행동**: 첫 화면은 "괜찮은가 / 뭘 해야 하나" 한 줄 + 액션 하나.
- **거짓말하지 않는다**: 건강 신호는 실제 차단/승인/폴백/미러 상태에서 도출(가짜
  green 없음). top action도 실제 nextActions 우선순위에서.
- **회귀 0**: 모든 기존 카드/CTA/deep-link는 펼치면 그대로. 테스트는 본문 검증을
  defaultDetailsOpen으로, 접힘(L1)은 새 테스트로 분리.
